## はじめに

この記事では、Godot 4.3とRust GDExtensionを使用してVR対応の3Dアプリケーションを開発する方法を詳しく解説します。OpenXRを活用した本格的なVR体験から、デスクトップでの通常操作まで、シームレスに対応する実装方法をご紹介します。

## 前提条件

- Godot 4.3以降
- Rust環境（cargo、rustc）
- VRヘッドセット（Meta Quest、HTC Vive、Index等のOpenXR対応デバイス）
- 基本的なGodotとRustの知識

## プロジェクト構成

```bash
meetia/
├── godot/
│   ├── project.godot          # プロジェクト設定
│   ├── scene.tscn             # デスクトップ用シーン
│   ├── vr_scene.tscn         # VR用シーン
│   └── openxr_action_map.tres # VRコントローラーアクション定義
└── rust/
    └── src/
        ├── lib.rs            # GDExtensionエントリーポイント
        ├── player.rs         # デスクトップ用プレイヤー
        └── vr_player.rs      # VRプレイヤーコントローラー
```

## 1. プロジェクト設定でVR機能を有効化

まず、Godotプロジェクトの設定ファイル `project.godot` でOpenXRを有効化します：

```ini
[application]
config/name="VR Application"
run/main_scene="res://vr_scene.tscn"
config/features=PackedStringArray("4.3", "Forward Plus")

[xr]
openxr/enabled=true
openxr/default_action_map="res://openxr_action_map.tres"
```

## 2. VRアクションマップの設定

VRコントローラーの入力を定義する `openxr_action_map.tres` を作成します：

```gdscript
[gd_resource type="OpenXRActionMap" format=3 uid="uid://bw5n2p8xq1kvs"]

[resource]
action_sets = Array[OpenXRActionSet]([SubResource("OpenXRActionSet_1")])
interaction_profiles = Array[OpenXRInteractionProfile]([SubResource("OpenXRInteractionProfile_1")])

[sub_resource type="OpenXRActionSet" id="OpenXRActionSet_1"]
resource_name = "default"
localized_name = "Default action set"
actions = Array[OpenXRAction]([
    SubResource("OpenXRAction_1"), # aim_pose
    SubResource("OpenXRAction_2"), # grip_pose
    SubResource("OpenXRAction_3"), # select
    SubResource("OpenXRAction_4")  # move
])

[sub_resource type="OpenXRAction" id="OpenXRAction_4"]
resource_name = "move"
localized_name = "Move"
action_type = 1  # Vector2 input for thumbstick
```

このアクションマップにより、VRコントローラーのサムスティック（move）、トリガー（select）、ポーズ情報（aim_pose、grip_pose）が利用できます。

## 3. VRシーン構造の設計

VR用のシーンファイル `vr_scene.tscn` を作成します：

```gdscript
[gd_scene format=3 uid="uid://bxl8k5d6x2nw7"]

[node name="Main" type="Node3D"]

[node name="XROrigin3D" type="XROrigin3D" parent="."]

[node name="XRCamera3D" type="XRCamera3D" parent="XROrigin3D"]
transform = Transform3D(1, 0, 0, 0, 1, 0, 0, 0, 1, 0, 1.8, 0)

[node name="LeftHand" type="XRController3D" parent="XROrigin3D"]
tracker = &"left_hand"
pose = &"aim"

[node name="RightHand" type="XRController3D" parent="XROrigin3D"]
tracker = &"right_hand"
pose = &"aim"

[node name="DirectionalLight3D" type="DirectionalLight3D" parent="."]

[node name="OnlineGltfLoader" type="OnlineGltfLoader" parent="."]

[node name="VRPlayer" type="VRPlayer" parent="."]
```

### シーン構造のポイント

- **XROrigin3D**: VR空間の原点。プレイヤーの実世界での位置に対応
- **XRCamera3D**: VRヘッドセットの視点。高さ1.8mに設定（平均的な目の高さ）
- **XRController3D**: 左右のVRコントローラー。`tracker`でどちらの手かを指定

## 4. RustでVRプレイヤーコントローラーを実装

メインとなるVRプレイヤーコントローラー `vr_player.rs` を実装します：

```rust
use godot::classes::{INode3D, Node3D, XrController3D, XrInterface, XrServer};
use godot::prelude::*;

#[derive(GodotClass)]
#[class(base=Node3D)]
pub struct VRPlayer {
    speed: f32,
    smooth_turn_speed: f32,
    teleport_distance: f32,
    xr_interface: Option<Gd<XrInterface>>,
    base: Base<Node3D>,
}
```

### VR初期化処理

```rust
fn initialize_vr(&mut self) -> bool {
    let mut xr_server = XrServer::singleton();

    // OpenXRインターフェースを探す
    let interface_name = GString::from("OpenXR");
    if let Some(mut interface) = xr_server.find_interface(&interface_name) {
        self.xr_interface = Some(interface.clone());

        // インターフェースを初期化
        if interface.initialize() {
            // プライマリインターフェースに設定
            xr_server.set_primary_interface(Some(&interface));
            godot_print!("VR interface initialized successfully");
            return true;
        }
    }
    false
}
```

### コントローラー入力の取得

```rust
fn get_controller_input(&self, controller_name: &str) -> Vector2 {
    if let Some(origin) = self.get_xr_origin() {
        if let Some(controller_node) = origin.find_child(controller_name) {
            if let Ok(controller) = controller_node.try_cast::<XrController3D>() {
                // サムスティック入力を取得
                return controller.get_vector2("move");
            }
        }
    }
    Vector2::ZERO
}
```

### 移動処理の実装

```rust
fn handle_movement(&mut self, delta: f64) {
    let left_stick = self.get_controller_input("LeftHand");
    let right_stick = self.get_controller_input("RightHand");

    if let Some(mut origin) = self.get_xr_origin() {
        // 左スティックで移動
        if left_stick.length() > 0.1 {
            let movement_speed = self.speed * delta as f32 * left_stick.length();

            // カメラの前方・右方向ベクトルを取得
            if let Some(camera_node) = origin.find_child("XRCamera3D") {
                if let Ok(camera) = camera_node.try_cast::<Node3D>() {
                    let transform = camera.get_global_transform();
                    let forward = -transform.basis.col_c();
                    let right = transform.basis.col_a();

                    // 水平面に投影
                    let mut horizontal_forward = forward;
                    horizontal_forward.y = 0.0;
                    horizontal_forward = horizontal_forward.normalized();

                    let mut horizontal_right = right;
                    horizontal_right.y = 0.0;
                    horizontal_right = horizontal_right.normalized();

                    // 移動ベクトルを計算
                    let movement = horizontal_forward * left_stick.y + horizontal_right * left_stick.x;
                    let movement_vector = movement * movement_speed;

                    // XROriginに移動を適用
                    let current_pos = origin.get_position();
                    origin.set_position(current_pos + movement_vector);
                }
            }
        }

        // 右スティックで回転
        if right_stick.x.abs() > 0.1 {
            let rotation_speed = self.smooth_turn_speed * delta as f32;
            let rotation_amount = -right_stick.x * rotation_speed;
            origin.rotate_y(rotation_amount);
        }
    }
}
```

### テレポート機能

```rust
fn handle_teleportation(&mut self) {
    // 右コントローラーのトリガーでテレポート
    if self.is_button_pressed("RightHand", "select") {
        if let Some(mut origin) = self.get_xr_origin() {
            if let Some(right_hand_node) = origin.find_child("RightHand") {
                if let Ok(right_hand) = right_hand_node.try_cast::<XrController3D>() {
                    // コントローラーから前方にレイを飛ばす
                    let transform = right_hand.get_global_transform();
                    let ray_origin = transform.origin;
                    let ray_direction = -transform.basis.col_c();

                    // テレポート先を計算
                    let teleport_target = ray_origin + ray_direction * self.teleport_distance;

                    // 地面レベル（y=0）に投影
                    let mut target_position = origin.get_position();
                    target_position.x = teleport_target.x;
                    target_position.z = teleport_target.z;

                    origin.set_position(target_position);
                }
            }
        }
    }
}
```

## 5. VR/デスクトップモードの自動切り替え

GLTFローダーでVRとデスクトップモードを自動判定し、適切なプレイヤーコントローラーを選択する実装：

```rust
// VRモードかデスクトップモードかを検出
let xr_server = XrServer::singleton();
let is_vr_active = if let Some(primary_interface) = xr_server.get_primary_interface() {
    primary_interface.is_initialized()
} else {
    false
};

if is_vr_active {
    // VRモードの場合、GLTFモデルを前方に配置
    if let Ok(mut node3d_scene) = corrected_scene.clone().try_cast::<Node3D>() {
        node3d_scene.set_position(Vector3::new(0.0, 0.0, -2.0));
        corrected_scene = node3d_scene.upcast::<Node>();
    }
    self.base_mut().add_child(&corrected_scene.upcast::<Node>());
    godot_print!("GLTF model loaded in VR mode");
} else {
    // デスクトップモードの場合、従来のPlayer3Dを使用
    let mut player_controller = crate::player::Player3D::new_alloc();
    player_controller.bind_mut().set_speed(3.0);

    // GLTFモデルをプレイヤーコントローラーの子として追加
    player_controller.add_child(&corrected_scene.upcast::<Node>());
    self.base_mut().add_child(&player_controller.upcast::<Node>());

    godot_print!("GLTF model loaded in desktop mode");
}
```

## 6. VRコントローラー操作方法

実装されたVRコントローラーの操作方法：

### 基本操作

- **左サムスティック**: 移動（前後左右）
- **右サムスティック**: なめらかな回転
- **右トリガー**: テレポート移動

### 移動の特徴

- **方向**: ヘッドセットの向きに基づいた相対移動
- **速度**: サムスティックの傾きに応じて可変
- **水平移動**: Y軸（高さ）は固定で水平面のみ移動

## 7. ビルドと実行

### Rustライブラリのビルド

```bash
# 開発用ビルド
task build

# リリース用ビルド
task build-release

# フォーマット＋リント＋ビルド＋テスト
task all
```

### Godotでの実行

1. VRヘッドセットを接続・起動
2. SteamVRやOculus Link等のVRランタイムを開始
3. Godotエディターで `vr_scene.tscn` を開く
4. 「プロジェクトを実行」でVRアプリケーションが起動

## 8. トラブルシューティング

### よくある問題と対処法

**VRヘッドセットが認識されない**

- OpenXRランタイムが正しくインストールされているか確認
- SteamVRまたはOculus Linkが起動しているか確認
- Godotプロジェクト設定でOpenXRが有効になっているか確認

**コントローラー入力が反応しない**

- アクションマップの設定を確認
- コントローラーのバッテリー残量を確認
- VRランタイムでコントローラーが正しく認識されているか確認

**移動やテレポートが機能しない**

- XROrigin3Dのシーン階層が正しく設定されているか確認
- VRPlayerがシーンに追加されているか確認
- コンソールログでVR初期化メッセージを確認

## まとめ

この実装により、以下の機能を持つVRアプリケーションが完成します：

- **シームレスなモード切り替え**: VRヘッドセット接続時は自動でVRモード
- **直感的なVR操作**: サムスティック移動、スムーズターン、テレポート
- **拡張可能な設計**: 新しいVR機能を簡単に追加可能
- **Rust+Godotの利点**: 高性能な処理とGodotの豊富なVR機能の組み合わせ

Godot 4.3のOpenXRサポートとRust GDExtensionを組み合わせることで、高性能で保守しやすいVRアプリケーションを開発することができます。この実装をベースに、より複雑なVR体験を構築していくことが可能です。
